There are a few common reasons why the keypoint drawings (lines or circles) might not appear over your camera feed:

Coordinate Scaling:
The pose detector (MoveNet) may return normalized coordinates (values between 0 and 1) rather than pixel values. In that case, you need to multiply the coordinates by the canvas (or video) width and height to correctly position the drawings.

Canvas Size Mismatch:
Ensure that the canvas size exactly matches the video element’s dimensions so that the drawn keypoints appear in the correct locations.

Insufficient Lighting or Incorrect Camera Angle:
Pose detectors rely on clear, well-lit images. If the camera feed is too dark or your lower body isn’t fully in view, the detector may not return reliable keypoints.

Model Configuration or Results:
There’s a chance that the model isn’t being fed the image correctly or the results aren’t being processed as expected. Logging the keypoints can help debug this.

const drawKeypoints = (keypoints: any, ctx: CanvasRenderingContext2D, width: number, height: number) => {
  keypoints.forEach((point: any) => {
    // Assume that if the point.x is less than 1, it's normalized.
    const x = point.x < 1 ? point.x * width : point.x;
    const y = point.y < 1 ? point.y * height : point.y;
    if (point.score > 0.3 || (point.visibility && point.visibility > 0.3)) {
      ctx.beginPath();
      ctx.arc(x, y, 6, 0, 2 * Math.PI);
      ctx.fillStyle = 'red';
      ctx.fill();
    }
  });
};
